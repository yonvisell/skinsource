SkinSource 2.0

A static browser application for predicting dynamic tactile signals in the hand and arm from the SkinSource impulse-response dataset.

Local-first: data and computation remain in the browser. SkinSource 2.0 facilitates rapid haptic validation without requiring new physical experiments.

Return to SkinSource 2.0

What the application calculates

SkinSource 2.0 convolves each assigned input acceleration signal with the measured three-axis SkinSource impulse responses, then superposes the resulting responses. The output is acceleration at 72 dorsal hand and forearm locations. All calculations run locally in the browser; no MATLAB installation or data upload is required.

The displayed acceleration quantity can be the skin-normal component, either local tangential component, the vector magnitude, or the per-location axis carrying the greatest RMS energy. The surface map reports RMS acceleration in decibels relative to the maximum value in the current response.

Basic workflow

  1. Inspect the default 100 Hz signal at Loc 7, or choose another upper-limb recording, input contact location, and signal.
  2. Select a row under Current input signals to edit it. Slider edits are applied to the response when the interaction is released or completed.
  3. To create another signal, set its parameters and select Add input. Use Replace to replace signals already assigned to that contact location.
  4. Add further input signals as needed. The response updates automatically.
  5. Select dorsal output locations on the surface response. Shift-click adds or removes outputs from the selection.
  6. Compare selected outputs in stacked or overlaid time and frequency plots.

Signals and views

  • Generated inputs include sinusoids, impulses, taps, and repeatable white noise.
  • WAV files are mixed to mono, resampled to 1300 Hz when necessary, and peak-normalized before application of the input amplitude gain.
  • The interpolated surface uses a portable, precomputed natural-neighbor interpolation operator matching the MATLAB visualization geometry.
  • Time plots show the selected acceleration quantity q(t) in m/s².
  • Frequency plots show the one-sided amplitude spectrum |FFT(q)| in m/s², with optional logarithmic frequency and magnitude axes.

Saving and exporting

CSV exports contain selected time responses, the primary selected output spectrum, or the complete RMS surface. The primary output can also be exported as a peak-normalized WAV file. Surface images use a black background and omit selected-output highlighting. Short surface-response movies are exported as WebM files.

A session JSON file preserves the input signals, model, displayed quantity, selected outputs, surface settings, and plot layout. Loading that file reconstructs the saved calculation without requiring the original WAV files.
